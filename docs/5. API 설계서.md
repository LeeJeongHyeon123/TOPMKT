## 시스템 개요

탑마케팅 플랫폼은 전통적인 MVC(Model-View-Controller) 패턴을 사용하는 PHP 웹 애플리케이션입니다.

- **아키텍처:** MVC 패턴
- **프론트 컨트롤러:** `/index.php`
- **라우팅:** `/src/config/routes.php`
- **세션 기반 인증:** PHP 세션 사용
- **뷰 렌더링:** PHP 템플릿 사용

## 요청/응답 형식

### HTML 응답 (페이지 렌더링)
대부분의 GET 요청은 HTML 페이지를 반환합니다:
```php
// 컨트롤러 예시
public function index() {
    $posts = $this->postModel->getList();
    $this->view('community/index', ['posts' => $posts]);
}
```

### JSON 응답 (AJAX API)
AJAX 요청에 대해서는 JSON 형식으로 응답합니다:
```php
// 성공 응답
{
    "success": true,
    "message": "작업이 완료되었습니다.",
    "data": {
        "id": 123,
        "title": "제목"
    }
}

// 에러 응답
{
    "success": false,
    "message": "오류가 발생했습니다.",
    "errors": {
        "field": "에러 메시지"
    }
}
```

### 폼 데이터 처리
POST 요청은 주로 `application/x-www-form-urlencoded` 또는 `multipart/form-data` 형식을 사용합니다:
```php
// 폼 데이터 예시
title=게시글 제목&content=내용&csrf_token=토큰값
```

## 인증 및 보안

### 세션 기반 인증
- PHP 세션을 사용한 사용자 인증
- 로그인 시 세션에 사용자 정보 저장
- 각 요청마다 세션 확인

```php
// 인증 확인 예시
if (!isset($_SESSION['user'])) {
    // 로그인 페이지로 리다이렉트
}
```

### CSRF 보호
- 모든 POST/PUT/DELETE 요청에 CSRF 토큰 필요
- 세션에 토큰 저장 및 검증

```php
// CSRF 토큰 생성
$_SESSION['csrf_token'] = bin2hex(random_bytes(32));

// 폼에 토큰 포함
<input type="hidden" name="csrf_token" value="<?= $_SESSION['csrf_token'] ?>">
```

### 자동 로그인 (Remember Me)
- 쿠키 기반 자동 로그인 기능
- `remember_token` 사용

## 에러 처리

### HTTP 상태 코드
| 코드 | 의미 | 사용 예시 |
|------|------|-----------|
| 200 | 성공 | 정상 처리 |
| 201 | 생성됨 | 리소스 생성 성공 |
| 302 | 리다이렉트 | 페이지 이동 |
| 400 | 잘못된 요청 | 유효성 검사 실패 |
| 401 | 인증 필요 | 로그인 필요 |
| 403 | 권한 없음 | 접근 권한 없음 |
| 404 | 찾을 수 없음 | 리소스 없음 |
| 500 | 서버 오류 | 처리 중 오류 |

### 에러 페이지
- `/src/views/templates/404.php` - 404 에러 페이지
- `/src/views/templates/403.php` - 403 에러 페이지

### 에러 로깅
- 모든 에러는 `error_log()`를 통해 기록
- 디버그 모드에서는 화면에 표시
- 스택 추적 정보 포함 (v3.2.0부터)

### ResponseHelper 표준화 (v3.2.0)
**표준 JSON 응답 형식**:
```php
ResponseHelper::json($data, $httpStatusCode, $message)
```

**응답 구조**:
```json
{
    "status": "success|error",
    "data": {...},
    "message": "응답 메시지",
    "timestamp": "2025-07-07T10:30:00+09:00",
    "request_id": "req_123456"
}
```

**주요 개선사항**:
- 파라미터 순서 표준화: `json($data, $code, $message)`
- 모든 API 엔드포인트 통일된 응답 형식 적용
- 요청 ID 자동 생성으로 디버깅 향상
