- `email` (varchar(100)): 이메일 (선택 사항)

   ```javascript
   if (!firebase.apps.length) {
     firebase.initializeApp(firebaseConfig);
   }
   ```
3. 초기화 순서:
   - Firebase Core
   - Authentication
   - Firestore
   - Storage
4. 초기화 실패 시 로깅 필수

## 스토리지 사용 규칙

본 프로젝트는 아래 3가지 저장소를 목적별로 분리하여 사용합니다:

1. **MariaDB**  
   - 사용자, 게시글, 댓글, 좋아요 등 **정형적 관계형 데이터 저장**

2. **Firestore**  
   - 채팅, 알림, 상태변경 등 **실시간 동기화 데이터 저장**

3. **Firebase Storage**  
   - 이미지, 영상, 첨부파일 등 **대용량 파일 저장소**  
   - 업로드 후 **해당 파일의 다운로드 URL을 MariaDB 또는 Firestore에 저장**

### ⚠️ 스토리지 선택 기준 요약
- **회원/게시글/댓글/좋아요 등** → MariaDB
- **채팅/알림 등 실시간 요소** → Firestore
- **이미지/파일 등 첨부자료** → Firebase Storage (→ URL만 DB에 저장)

> ⚠️ 반드시 이 기준을 따라 스토리지를 선택해야 하며,  
> 설계 및 선택은 AI가 최적 판단하여 자동 생성/관리함.  
> **Realtime Database는 사용하지 않음**

## 백업 정책
1. **백업 프로세스**
   - 백업 요청 시 다음 두 가지를 모두 수행:
     1. Git 백업: 현재 상태를 Git에 커밋 및 푸시
     2. 전체 소스 백업: 프로젝트 폴더 전체를 `/backup/git/topmkt_YYYYMMDD_HHMMSS`로 복사
   - 백업 스크립트 위치: `./shScript/manual_backup.sh`
   - 검증 스크립트 위치: `./shScript/verify_backup.sh`
   - 백업 실행 방법:
     ```bash
     ./shScript/manual_backup.sh
     ```
   - 백업 검증 방법:
     ```bash
     ./shScript/verify_backup.sh /backup/git/topmkt_YYYYMMDD_HHMMSS
     ```

2. **Git 백업**
   - 프로젝트 전체 소스코드는 `/backup/git/topmkt_YYYYMMDD_HHMMSS` 형식으로 백업
   - 백업 시점: 주요 기능 개발 완료 시점
   - 백업 방법: 전체 프로젝트 폴더 복사
   - 백업 정보 파일: `backup_info.txt`에 다음 정보 포함
     - 백업 시간
     - 백업 유형
     - 백업 크기
     - GIT 태그
     - 커밋 메시지
     - 태그 메시지
   - 복구 방법: 
     - 최신 백업: `/backup/git/topmkt_YYYYMMDD_HHMMSS`의 내용을 `/var/www/html/topmkt`로 복사
     - 특정 시점 복구: 해당 시점의 백업 폴더를 선택하여 복사

3. **백업 검증 프로세스**

   - 테스트 데이터 초기화 스크립트 위치: `/scripts/cleanup_user.sh`
   - 초기화 대상:
     - MariaDB의 users 테이블에 저장된 사용자 정보
     - Firebase Authentication에 등록된 사용자 계정

   - 모든 테스트 사용자 삭제:
     ```bash
     cd /var/www/html/topmkt/scripts && ./cleanup_user.sh
     ```

     ```bash
     cd /var/www/html/topmkt/scripts && ./cleanup_user.sh 01012345678
     ```
