# 기본 정책 문서
- 프로젝트(서비스) 이름: 탑마케팅 (TOPMKT)

## 응답 규칙
- 모든 응답은 반드시 **한국어**로 작성할 것
- **명령어 실행 시 이 문서를 항상 참조**할 것

## 프로젝트 개발 규칙
- **`tail` 명령어 사용금지**
- .htaccess 파일을 통해 URL 리라이팅을 설정하여 .php 확장자를 숨김.
- 만들어진 로고 이미지 파일이 있음. 나중에 업로드 후 경로 알려주겠음. 그전까지 TEXT로 로그처럼 만들어서 진행.
- 웹 서버에서 `탑마케팅` 프로젝트 외에 **다른 웹 서비스가 같이 가동 중**임을 인지하고 다른 서비스에 문제를 발생 시키는 작업을 하면 안 되고 꼭 해야 하는 게 있다면 나한테 검토 받고 물어보고 난 후에 진행
- OS 명령어 mysql 사용시 패스워드는 AI가 직접 입력. 사용자한테 입력 받지 않고 진행
- Git 커밋 및 푸시는 AI가 **적절한 타이밍에 자동으로 수행**
- 커밋 메시지는 **반드시 한국어로 작성**
- **모든 로직에 상세한 한국어 주석 작성 필수**
- **보안 설정 변경 시 반드시 사용자 확인 요청**
- 본 프로젝트는 **반응형 웹 기반 PHP 프로젝트**이며, **정적 HTML 파일은 사용하지 않음**
- 백엔드 언어: PHP 7.4.33
- 웹 루트 디렉토리: `/var/www/html/topmkt`
- 웹 서버 OS: CentOS Linux release 7.9.2009 (Core)
- 웹 서버: Apache/2.4.58
- 작업 경로는 **SSH를 통해 루트 디렉토리 기준으로 진행**
- 웹 폰트는 URL 방식이 아닌 **직접 다운로드 후 서버에 저장하여 사용**
- 서비스는 기본 언어로 **한국어**, 추가로 **영어, 중국어(간체/번체), 일본어** 지원되도록 설계
- 로그인 세션 타임아웃 설정 하지 않음. 무제한 세션임. 유저 편의를 위해.

## MariaDB 정보
- MariaDB 버전: 10.4.34
- 사용 스키마: `TOPMKT` (**해당 스키마만 사용 허용**)
- **다른 스키마(예: `WINCard`)는 select, update, delete 포함 일절 접근 금지**

## 스토리지 사용 규칙

본 프로젝트는 아래 3가지 저장소를 목적별로 분리하여 사용합니다:

1. **MariaDB**  
   - 사용자, 게시글, 댓글, 좋아요 등 **정형적 관계형 데이터 저장**

2. **Firestore**  
   - 채팅, 알림, 상태변경 등 **실시간 동기화 데이터 저장**

3. **Firebase Storage**  
   - 이미지, 영상, 첨부파일 등 **대용량 파일 저장소**  
   - 업로드 후 **해당 파일의 다운로드 URL을 MariaDB 또는 Firestore에 저장**

### ⚠️ 스토리지 선택 기준 요약
- **회원/게시글/댓글/좋아요 등** → MariaDB
- **채팅/알림 등 실시간 요소** → Firestore
- **이미지/파일 등 첨부자료** → Firebase Storage (→ URL만 DB에 저장)

> ⚠️ 반드시 이 기준을 따라 스토리지를 선택해야 하며,  
> 설계 및 선택은 AI가 최적 판단하여 자동 생성/관리함.  
> **Realtime Database는 사용하지 않음**

## 디렉토리 구조
- 프로젝트 디렉토리 구조는 `디렉토리구조.md` 파일에 정의됨
- 유저 페이지 헤더 파일: `/var/www/html/topmkt/includes/header.php`
- 유저 페이지 푸터 파일: `/var/www/html/topmkt/includes/footer.php`
- 헤더, 푸터는 모든 유저 페이지 UI에 헤더와 푸터로 인클루드 해서 사용.

## 다국어 번역 처리 방식: 파일 기반 언어 분리
- 경로: `/resources/lang/{언어코드}/`  
- PHP 배열 파일로 각 언어 파일 구성
- 키 기반 번역 구조 유지 (예: `messages.welcome`, `errors.not_found`)
- 사용 예: `trans('messages.welcome')` 또는 `__('messages.welcome')`

## 프론트엔드 언어 선택 UI 기준

- 현재 선택된 언어는 시각적으로 강조
- **언어명 병기**

## 인증 및 회원 기능 (Firebase Authentication 기반)
- **휴대폰 번호 인증 기반 로그인 및 회원가입 구현**
- 인증된 전화번호 = 로그인 ID (비밀번호 없음)
- 로그인 시 DB에 로그인 이력 기록 (ID, 시간, IP 등)
- 한 번 로그인하면 로그아웃 전까지 세션 유지(세션 시간 무제한한)
- **인증번호 1시간 내 5회 오류 → 24시간 인증 차단**  
  → 사용자에게 적절한 alert 안내 필요
- 테스트용 번호: `+82 10-1234-1234`, 인증코드: `123456`
