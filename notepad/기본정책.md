## Git 관리 정책

### 저장소 구조
- 메인 프로젝트: https://github.com/LeeJeongHyeon123/TOPMKT.git
- MCP 저장소: https://github.com/GLips/Figma-Context-MCP.git

### 브랜치 전략
- 기본 브랜치: master
- 브랜치 사용: 단일 master 브랜치 사용 (다중 브랜치 사용 지양)
- 작업 방식: master 브랜치에서 직접 작업

### MCP 관련
- MCP 디렉토리: /var/www/html/topmkt/mcp
- MCP 설정 파일: .env, package.json, tsconfig.json, src/
- MCP Git 연결: MCP 전용 저장소 사용

### 커밋 메시지 규칙
- 형식: [카테고리] 작업내용
- 카테고리: 
  - FEAT: 새로운 기능
  - FIX: 버그 수정
  - DOCS: 문서 수정
  - STYLE: 코드 포맷팅
  - REFACTOR: 코드 리팩토링
  - TEST: 테스트 코드
  - CHORE: 빌드 업무 수정

### Git 태깅 정책
- 모든 커밋은 태그와 함께 진행
- 태그 형식: v[주].[일].[순번]
  - 예: v1.0.1, v2.3.5
- 태그 생성 규칙:
  - 주: 주요 버전 변경 (기능 대폭 추가/변경)
  - 일: 기능 추가/수정
  - 순번: 버그 수정 및 소소한 변경
- 태그 메시지 형식: [카테고리] 태그 설명
  - 예: [FEAT] 사용자 인증 기능 추가
- 태그 생성 명령어:
  ```bash
  git tag -a v1.0.1 -m "[카테고리] 태그 설명"
  git push origin v1.0.1
  ```

### 주의사항
- MCP 관련 변경사항은 반드시 MCP 저장소에 커밋
- 메인 프로젝트와 MCP 저장소는 별도로 관리
- 브랜치 생성 시 불필요한 브랜치 생성을 지양 

## 헤더 메뉴 정책 및 구성

### 헤더 메뉴 구성 원칙
- 헤더 메뉴는 기획서 및 정책에 따라 고정적으로 유지한다.
- 메뉴 추가/삭제/변경 시 반드시 본 정책에 기록 후 적용한다.
- 메뉴 순서와 노출 조건(로그인 여부 등)도 명확히 기록한다.

### 현재 헤더 메뉴 구성(2024-06 기준)
1. 추천 리더
2. 회사/비전 소개
3. 노하우 공유
4. 팀 리쿠르팅 모집
5. 행사 일정
6. 강의 일정
7. 자유 커뮤니티
8. 공지사항

- 홈 메뉴는 헤더에서 제외함.
- 채팅 메뉴(읽지 않은 메시지 배지 포함)는 로그인 시에만 노출.
- 로그인/회원가입 버튼은 로그인 버튼 하나로 통합(로그인/회원가입이 한 페이지에 있으므로).
- 언어 선택 드롭다운은 우측에 위치.
